2. Write a short note on file based storage system. Explain the major challenges of file based system

File based storage system is a traditional approch to storing and managing data, where data is stored in individual files on a computer's file system. each file contains a collection of records with each record representing an entity or object in the sysytem.

Major challenges

  1. Data redundancy
  2. Data Inconsistency
  3. Security
  4. Data isolation

3. What is DBMS? what was the need for DBMS?

Database Management system are software systems used to store, retrieve and run queries on data. A DBMS serves as an interface between an end-user and a database, allowing users to create, read, update, abd delete data in the database.

Need of DBMS

  • Creation of a database
  • Retrieval of information from the database
  • Updating the database
  • Managing a database

The data model defines the physical and logical structure of a database which involves th datatypes, the relationship among the data,constraints applied on the data and even the basic operations specifying retrieval and updation of data in the database.

The database management systems can also be classifid on the basis of its user. so, a DBMS can either b used by a single user or it can be used by multiple users.

Depending on the distribution of the database over numerous sites we can classify the database as:

  • Centralized DBMS

    • In the Centralized DBMS, the entire database is stored in a single computer site.

  • Distributed DBMS

    • In the Distributed DBMS the database and the DBMS software are distributed over many omputer sites.

6. What is the significance of Data modelling and explain the types of Data modelling.

Data modeling is the process of creating a visual representation of either a whole information system or parts of it to communicate connections between data points and structures. The goal is to illustrate the types of data used and stored within the system, the relationship among these datatypes, the ways the data can be grouped and organised and its formats and attributes.

Types of Data modeling

  • Hierarchical data models
  • Relational data models
  • Entity-relationship data models
  • Object-oriented data models
  • Dimensional data models

7. Explain 3 schema architecture along with its advantages.

The three-schema architecture is a design approch for databases that splits data views into three layers.

Three schema architecture components:

  • External schema(user view)

    • This level represents the way individual users or user groups view the data.

  • Conceptual schema(logcal view)

    • This level represents the logical structure of the whole database as seen by the administrator.

  • Internal schema(physical view)

    • This level defines how and where the data is stored and accessed.

Advantages of three schema architecture

  • Data independence
  • Flexibility
  • Efficient data access
  • Improved security
